# 游戏活动运营数据小助手

一个面向游戏活动运营场景的静态网页工具，用于快速录入活动数据、自动计算关键运营指标，并生成可用于复盘的分析建议。

## 项目定位

本项目适合用于游戏活动上线后的初步效果评估，帮助运营人员把曝光、参与、完成、领奖、回访和成本数据转换为可读的指标看板与复盘结论。

## 核心功能

- 活动基础信息录入：活动名称、活动类型、目标用户、活动目标和活动周期。
- 活动数据录入：曝光人数、参与人数、完成人数、奖励领取人数、次日回访人数和活动总成本。
- 热门游戏活动模板：内置原神、王者荣耀、和平精英、崩坏：星穹铁道、明日方舟、阴阳师、蛋仔派对、第五人格等模拟活动模板。
- 指标自动计算：参与率、完成率、奖励领取率、次日回访率和人均活动成本。
- 综合评级：根据核心指标输出 A/B/C/D 活动评级和解释。
- 自动复盘报告：生成活动基本情况、数据表现总结、指标分析和优化建议。
- 数据导入：支持 `.xlsx`、`.csv`、`.md`、`.txt` 文件导入，也支持从 Excel/WPS 或 Markdown 表格粘贴导入。
- 异常校验：检测空值、负数、分母为 0，以及参与人数大于曝光人数等逻辑异常。

## 快速开始

本项目不依赖构建工具，直接打开 `index.html` 即可使用。

### 本地打开

1. 下载或克隆项目。
2. 双击打开 `index.html`。
3. 点击“填入示例数据”或选择热门游戏活动模板。
4. 点击“生成复盘”查看指标看板、活动评级和复盘报告。

### 本地静态服务

如果需要通过本地服务预览，可以在项目目录运行：

```powershell
python -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

## 数据字段说明

| 字段 | 含义 | 用途 |
| --- | --- | --- |
| 曝光人数 | 看到活动入口或活动信息的人数 | 计算参与率 |
| 参与人数 | 实际进入或参与活动的人数 | 计算参与率、完成率、回访率和人均成本 |
| 完成人数 | 完成活动目标任务的人数 | 计算完成率和奖励领取率 |
| 奖励领取人数 | 完成后领取奖励的人数 | 计算奖励领取率 |
| 次日回访人数 | 参与后次日再次访问或回到游戏的人数 | 计算次日回访率 |
| 活动总成本 | 活动奖励、资源或投放成本 | 计算人均活动成本 |

## 指标口径

| 指标 | 公式 | 说明 |
| --- | --- | --- |
| 参与率 | 参与人数 / 曝光人数 | 衡量活动入口曝光和奖励吸引力 |
| 完成率 | 完成人数 / 参与人数 | 衡量任务难度和活动流程合理性 |
| 奖励领取率 | 奖励领取人数 / 完成人数 | 衡量奖励领取路径是否清晰顺畅 |
| 次日回访率 | 次日回访人数 / 参与人数 | 衡量活动对后续活跃的带动效果 |
| 人均活动成本 | 活动总成本 / 参与人数 | 衡量单个参与玩家的平均投入成本 |

## 技术说明

- 前端结构：原生 HTML、CSS、JavaScript。
- 样式主题：深色战术控制台风格，主色为薄荷绿，点缀色为琥珀橙。
- 数据处理：浏览器端完成，无后端服务。
- Excel 解析：使用浏览器原生能力解析 `.xlsx` 的 ZIP/XML 结构，不依赖第三方库。
- 部署方式：可部署到 GitHub Pages、Vercel、Netlify 或任意静态文件服务。

## 注意事项

- 内置热门游戏活动模板为模拟运营场景数据，不代表真实线上数据。
- 当前复盘报告是基于固定阈值和规则生成，适合作为活动复盘初稿。
- 若用于正式经营分析，建议结合 DAU、ARPU、付费转化率、留存曲线、活动分层人群等更多数据一起判断。
